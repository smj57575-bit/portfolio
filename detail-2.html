OB맥주 FAQ 페이지

📅 2026년 2월 👤 역할: UI/UX 퍼블리셔

프로젝트 개요

OB맥주 홈페이지 내 FAQ 페이지를 신규 구축한 프로젝트로, 단순한 페이지 제작을 넘어 SEO 및 AEO를 고려한 콘텐츠 구조 설계에 집중했습니다.

사용자의 검색 의도를 반영한 질문-답변(Q&A) 형식으로 콘텐츠를 구성하고, 검색엔진 친화적인 마크업을 적용해 검색 노출과 정보 탐색 효율을 함께 높이는 방향으로 작업했습니다.

주요 역할 및 업무

• FAQ 콘텐츠 구조 설계
사용자 검색 의도를 반영한 질문-답변(Q&A) 구조를 설계하여, 필요한 정보를 빠르게 찾을 수 있도록 콘텐츠 흐름을 구성

• SEO / AEO 최적화 마크업 적용
검색엔진 친화적인 시맨틱 마크업과 메타 정보 구성을 통해 검색 노출 효율을 높이고, AI 검색 환경에서도 인용 가능한 구조로 정리

• 신규 페이지 퍼블리싱
OB맥주 홈페이지 톤앤매너에 맞춰 FAQ 페이지를 신규 구축하고, 다양한 디바이스에서 자연스럽게 동작하도록 화면 구조 구현

• 이미지 최적화 및 네이밍 전략 적용
페이지 내 이미지를 WebP 포맷으로 변환해 용량을 최적화하고, 단순 파일명이 아닌 이미지 설명 기반 네이밍을 적용해 검색엔진 및 AI 노출 관점의 문맥 인식까지 고려

• 검색 성과 검증
SEO 품질 점검을 통해 95점 이상의 평가 점수를 달성하고, 실제 AI 검색 결과 노출까지 확인

기술 스택

HTML5 CSS3 Schema SEO Figma

주요 코드 미리보기

※ 실제 운영 파일을 그대로 공개하는 대신, 핵심 구현 방식을 일반화한 예시 코드입니다. (사내 식별자·도메인·업로드 경로는 대표 예시로 대체)

1) 헤더 공통 SEO/스키마 구성

페이지 유형에 따라 FAQ 스키마를 분기 출력하도록 설계해, 각 페이지 목적에 맞는 구조화 데이터가 자동으로 적용되도록 코드 구조를 구성했습니다.

<?php if (isExact('/about/faq.php', $N)): ?>
  <script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@type": "FAQPage",
    "name": "기업 및 브랜드 FAQ",
    "mainEntity": [
      { "@type": "Question", "name": "회사는 어떤 배경에서 시작되었나요?" },
      { "@type": "Question", "name": "기업 비전과 슬로건은 무엇인가요?" }
    ]
  }
  </script>
  <script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@type": "BreadcrumbList",
    "itemListElement": [
      { "@type": "ListItem", "position": 1, "name": "홈", "item": "https://example.com" },
      { "@type": "ListItem", "position": 2, "name": "FAQ", "item": "https://example.com/about/faq.php" },
      { "@type": "ListItem", "position": 3, "name": "기업·브랜드", "item": "https://example.com/about/faq.php" }
    ]
  }
  </script>
<?php elseif (isExact('/sustainability/faq.php', $N)): ?>
  <script type="application/ld+json">{ "@type": "FAQPage", "name": "지속가능경영(ESG) FAQ", ... }</script>
  <script type="application/ld+json">{ "@type": "BreadcrumbList", ... }</script>
  <script type="application/ld+json">{ "@type": "CollectionPage", ... }</script>
<?php endif; ?>

2) FAQ 시맨틱 마크업 + 구조화 데이터 결합

질문/답변 콘텐츠를 Question/Answer 구조화 데이터와 함께 설계해, 검색엔진이 문답 맥락을 명확하게 이해할 수 있도록 구성한 패턴입니다.

<article
  class="faq-item"
  itemscope
  itemprop="mainEntity"
  itemtype="https://schema.org/Question"
>
  <div
    class="faq-question"
    onclick="toggleFaq(this)"
    tabindex="0"
    role="button"
    aria-expanded="false"
  >
    <h3 class="faq-h3" itemprop="name">
      지속가능경영을 어떻게 실천하고 있나요?
    </h3>
  </div>

  <div
    class="faq-answer"
    itemscope
    itemprop="acceptedAnswer"
    itemtype="https://schema.org/Answer"
  >
    <div class="faq-answer-content" itemprop="text">
      ... 답변 콘텐츠 ...
    </div>
  </div>
</article>

3) 질문/답변형 시맨틱 마크업 구조

AI 엔진이 콘텐츠 맥락을 정확히 파악할 수 있도록 h2, h3 heading 계층을 명확히 구성하고, Question/Answer 스키마를 함께 적용한 FAQ 문답형 구조입니다.

<main class="faq-page">
  <section id="company-history" class="faq-section" aria-labelledby="company-heading">
    <div class="section-header">
      <h2 class="faq-h2" id="company-heading">회사소개 · 역사 · 연혁</h2>
    </div>

    <div class="faq-container">
      <article class="faq-item"
               itemscope
               itemprop="mainEntity"
               itemtype="https://schema.org/Question">
        <div class="faq-question" role="button" aria-expanded="false">
          <h3 class="faq-h3" itemprop="name">
            회사는 어떤 배경에서 시작되었나요?
          </h3>
        </div>

        <div class="faq-answer"
             itemscope
             itemprop="acceptedAnswer"
             itemtype="https://schema.org/Answer">
          <div class="faq-answer-content" itemprop="text">
            ... 답변 콘텐츠 ...
          </div>
        </div>
      </article>
    </div>
  </section>
</main>
