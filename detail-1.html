📅 2026년 3월 ~ 진행중 👤 역할: UI/UX 퍼블리셔

프로젝트 개요

DB손해보험 디자인 리뉴얼 프로젝트로, 기존 적응형(Adaptive) 구조를 반응형(Responsive) 구조로 전환하는 작업을 담당했습니다.

단순 전환이 아닌, 기존 적응형 페이지는 계속 운영해야 하는 상황이었기 때문에 전체를 한 번에 바꾸는 것이 아닌 순차적 전환 방식으로 진행되었습니다. 이로 인해 적응형과 반응형 구조가 동시에 운영되는 환경에서, 전환 대상 페이지에 한해 기존 모바일/PC 분리 코드를 통합하는 방식으로 구조를 설계했습니다.

핵심 도전 과제

• 적응형·반응형 구조 동시 운영
기존 적응형 코드는 그대로 유지하면서, 반응형으로 전환되는 페이지들만 선별적으로 모바일/PC 코드를 통합해야 했습니다. 파일 구조 설계와 코드 정리에 많은 시행착오를 거쳐, 최종적으로 두 구조를 안정적으로 병행 운영할 수 있는 체계를 구축했습니다.

• 공통 컴포넌트 분리 및 파일 구조 재정비
기존 코드는 공통으로 사용 가능한 요소들이 각 페이지에 개별적으로 포함되어 있어 유지보수가 어려운 구조였습니다. 공통 컴포넌트를 분리하고 공통 파일 구조로 재정리하여 유지보수성을 크게 개선했습니다.

• 신규 보험 가이드 페이지 제작
리뉴얼과 함께 신규 보험 가이드 페이지를 기획·제작하여, SEO / AEO / GEO 관점에서 검색 노출 및 인용률 향상을 도모했습니다. 특히 보험 가이드 메뉴 내 driver-01 페이지에서 AI를 이용하여 스키마를 작성하고 연결했으며, 메타 태그·canonical·Open Graph와 함께 적용해 검색 친화적인 콘텐츠 구조를 구축했습니다.

주요 역할 및 업무

• 반응형 웹 퍼블리싱
모바일·태블릿·데스크톱 전 구간에서 자동으로 대응하는 유연한 레이아웃 구현

• 순차 전환 구조 설계
적응형 운영을 유지하면서 반응형 전환 페이지만 선별적으로 코드를 통합하는 파일 구조 및 분기 체계 설계

• SEO / AEO / GEO 마크업 강화
시맨틱 마크업, heading 계층 구조, AI를 이용한 스키마 작성 및 연결, meta 태그 정비를 통해 검색 노출 및 AI 인용률 개선

• 공통 컴포넌트 분리
페이지별로 중복 포함되어 있던 공통 요소를 별도 파일로 분리하고 구조를 재정비하여 유지보수 효율화

    placeholder="예) 19990101"

4) 보험 가이드 SEO, AEO 작업

가이드 상세 페이지에 AI를 이용해 스키마를 작성·연결하고, 검색/공유/AI 인용까지 고려한 메타 구성을 적용한 사례입니다.

<title>가이드 페이지 제목 ... | ○○보험</title>
<meta name="description" content="가이드 페이지 요약 설명 ...">
<meta name="keywords" content="키워드1, 키워드2, 키워드3">
<link rel="canonical" href="https://example.com/guide/guide-01">
<meta property="og:url" content="https://example.com/guide/guide-01">
<meta property="article:published_time" content="2026-03-25T00:00:00+09:00">

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "가이드 페이지 제목 ...",
  "datePublished": "2026-03-25",
  "dateModified": "2026-04-06"
}
</script>
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "position": 1, "name": "홈" },
    { "position": 2, "name": "가이드" }
  ]
}
</script>
<script type="application/ld+json">
{ "@context":"https://schema.org", "@type":"FAQPage", "mainEntity":[ ... ] }
</script>
<script type="application/ld+json">
{ "@context":"https://schema.org", "@type":"HowTo", "step":[ ... ] }
</script>
